You are a senior .NET quantitative developer.

I have an existing backtesting engine with a method RunORB.

Your task is to create a NEW strategy method:

RunOrbFvgStrategy

This should integrate cleanly into the existing system.

---------------------------------------

GOAL

Implement a high-quality intraday strategy based on:

- Opening Range Breakout (ORB)
- Fair Value Gap (FVG)
- Retracement
- Engulfing confirmation

This is a multi-step strategy.

---------------------------------------

STRATEGY LOGIC

STEP 1: OPENING RANGE

- Use first 5 candles (1-min) → 09:15–09:20
- Define:
    openingHigh
    openingLow

---------------------------------------

STEP 2: BREAKOUT DETECTION

After 09:20:

Bullish:
    close > openingHigh

Bearish:
    close < openingLow

IMPORTANT:

- Only CLOSE breakout
- Ignore wicks

---------------------------------------

STEP 3: STATE MACHINE (VERY IMPORTANT)

Implement a state-based flow:

States:

- WaitingForBreakout
- WaitingForFvg
- WaitingForRetrace
- WaitingForEngulfing
- TradeTriggered

---------------------------------------

STEP 4: FVG DETECTION

Use 3-candle structure:

Bullish:
    candle1.high < candle3.low

Bearish:
    candle1.low > candle3.high

Rules:

- Must form AFTER breakout
- Ignore small gaps:
    minimum gap = 0.1% price

---------------------------------------

STEP 5: RETRACE INTO FVG

- Wait for price to come back into FVG zone
- No chasing breakout

---------------------------------------

STEP 6: ENGULFING CONFIRMATION

Bullish:

- Current candle engulfs previous
- Close > previous high

Bearish:

- Current candle engulfs previous
- Close < previous low

---------------------------------------

STEP 7: TREND FILTER

Use EMA(20):

Bullish:
    price > EMA
    EMA rising

Bearish:
    price < EMA
    EMA falling

---------------------------------------

STEP 8: VOLUME FILTER

- Engulfing candle volume > average last 10 candles

---------------------------------------

STEP 9: ENTRY

- Entry at next candle OPEN

---------------------------------------

STEP 10: STOP LOSS

Bullish:
    below FVG low

Bearish:
    above FVG high

---------------------------------------

STEP 11: TARGET

- Risk:Reward = 3:1

---------------------------------------

STEP 12: TIME FILTER

- Trade only between 09:20–10:30 IST

---------------------------------------

STEP 13: RISK CONTROL

- Max 1 trade per day
- Skip low volatility days

---------------------------------------

STEP 14: OUTPUT

Return List<BacktestTradeResult>

Use existing:

- cost functions
- slippage
- quantity logic

---------------------------------------

STEP 15: CODE REQUIREMENTS

- Clean C#
- Reuse existing utilities
- Add helper methods:

    DetectFvg()
    IsEngulfing()
    IsTrendValid()

- Use clear variable names
- Comment each step

---------------------------------------

GOAL

This strategy should produce:

- Fewer trades
- Higher accuracy
- Better RR trades

Avoid overtrading.

Focus on precision.
